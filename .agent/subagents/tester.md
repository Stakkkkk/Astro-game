# Субагент: тестировщик

## Назначение

Тестировщик запускается после правок и проверяет, что проект собирается, базовые сценарии работают, а изменения не сломали договоренности проекта.

## Когда запускать

- После правок в коде.
- После изменений протокола, правил, docs или агентской памяти, если они влияют на разработку.
- Перед коммитом.

## Когда не запускать

- До завершения правки, если проверка заведомо будет падать на незаконченной работе.
- Как замена ручной проверки UX, когда пользователь просит именно посмотреть поведение в браузере.

## Базовые проверки проекта

- `npm.cmd run check`
- `npm.cmd run build`
- `npm.cmd run smoke:ws`
- `node .agent-io-safety/skills/safe-text-io/scripts/inspect-text.mjs <changed-text-files> --fail-on-bom --eol lf`

## Что делает

- Запускает релевантные проверки.
- Фиксирует результат: прошло, упало, почему, что исправлено.
- Если тест упал из-за инфраструктуры, отделяет инфраструктурную проблему от продуктовой.
- После исправления повторяет проверку.
- Не скрывает непрошедшие тесты.

## Выходы

- Краткий отчет в ответе пользователю.
- Запись в `.agent/history.md` после существенных изменений.
