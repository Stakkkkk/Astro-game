# Заметка к кейсу: тач-управление

Дата: `2026-07-04 13:46:18 +07:00`.

## Ситуация

Пользователь проверил публичную игру с мобильного устройства и сформулировал проблему простым продуктовым языком: «С мобилки сейчас не полетать. Нужен слева джойстик, справа кнопка стрельбы».

## Методическое наблюдение

- После появления публичной ссылки быстро возникают проверки с реальных устройств, которых не было в локальном desktop-сценарии.
- Для не ИТ-пользователя формулировка через ожидаемую игровую схему управления естественнее, чем через технические требования к pointer events или responsive layout.
- Агенту полезно переводить такую формулировку в минимальное изменение протокола. В этом случае сервер менять не нужно: touch controls можно свести к уже существующим `left/right/thrust/shoot`.

## Практический прием

- Не добавлять отдельный мобильный протокол, если можно объединить touch input и keyboard input на клиенте.
- Показывать touch controls только после входа в комнату, чтобы стартовая форма оставалась чистой.
- Прятать debug overlay на мобильном, если он перекрывает важные игровые кнопки.
- После UI-правки для мобильного обязательно перезаливать preview, иначе пользователь с телефона увидит старый build.
