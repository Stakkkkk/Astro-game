# ADR-002: авторитетная сетевая модель

## Статус

Принято.

## Контекст

Игра должна быть настоящей онлайн-игрой, а не локальной демкой с заглушкой сети. Несколько игроков должны видеть один общий мир, астероиды, снаряды и рейтинг.

## Решение

Сервер авторитетный:

- клиент отправляет только input;
- сервер считает движение, столкновения, урон и очки;
- сервер рассылает snapshot состояния комнаты;
- клиент отрисовывает полученное состояние.

## Параметры MVP

- tickrate: `20 Hz`;
- игроков в комнате: 2-8;
- мир: фиксированный тороидальный 2D-мир;
- частота input: около `20 Hz`;
- persistence: нет, комнаты живут в памяти процесса.

## Последствия

- Клиент не может сам начислить себе очки.
- Первый MVP проще отлаживать.
- Сложные техники вроде rollback и client-side prediction можно добавить позже.
