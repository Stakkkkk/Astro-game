# ADR-001: выбор платформы MVP

## Статус

Принято.

## Контекст

Проект должен быть учебной онлайн-игрой, которую легко открыть по ссылке и проверить в двух браузерах. Концепт просит сравнить минимум два направления: Web + TypeScript + WebSocket и online-first Cloudflare Pages/Workers/Durable Objects.

## Рассмотренные варианты

### Web + TypeScript + Node.js WebSocket server

Плюсы:

- быстро запускается локально;
- один язык на клиенте, сервере и shared-пакете;
- стандартная WebSocket-модель;
- игровой loop проще держать в памяти;
- легче объяснять человеку без ИТ-бэкграунда.

Минусы:

- для публичного онлайна позже нужен хостинг сервера;
- деплой сложнее, чем один статический сайт.

### Web + TypeScript + Cloudflare Workers/Durable Objects

Плюсы:

- хорошо подходит для online-first;
- можно связать с GitHub и Cloudflare;
- Durable Objects естественно похожи на комнаты.

Минусы:

- сложнее стартовая отладка;
- runtime имеет ограничения;
- WebSocket + game loop требуют аккуратной архитектуры;
- выше риск застрять до появления играбельной версии.

## Решение

Для первого MVP выбран Web + TypeScript + Node.js WebSocket server. После работающего локального онлайн-slice отдельно вернемся к деплою и сравним перенос комнат на Cloudflare Durable Objects с размещением Node.js backend.

## Последствия

- Первый запуск максимально простой: `npm install`, `npm run dev`.
- Серверный код остается понятным и наблюдаемым.
- Cloudflare не отвергается, но переносится на этап после доказанной игровой механики.
