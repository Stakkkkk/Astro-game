# Архитектура

## Решение для первого MVP

Первый вертикальный срез строится как TypeScript monorepo:

```text
apps/client    Browser + Canvas 2D + Vite
apps/server    Local Node.js + ws adapter
apps/worker    Cloudflare Worker + Durable Object + Static Assets
packages/game-core Shared authoritative room engine
packages/shared Protocol, DTO, constants, math helpers
```

Причина: такой стек быстрее всего дает настоящий онлайн-slice, понятен для учебного проекта и не требует отдельного игрового движка.

## Поток данных

```text
Keyboard
  -> Client input state
  -> WebSocket ClientMessage.input
  -> Authoritative Node.js server tick
  -> Server-side physics/collisions/scoring
  -> WebSocket ServerMessage.snapshot
  -> Canvas render + HUD
```

Клиент отправляет только намерения игрока. Сервер решает, где находится корабль, попал ли снаряд, разрушился ли астероид и сколько очков начислить.

## Игровой мир

Для первого MVP выбран очень большой тороидальный мир. Объекты, вылетевшие за границу, появляются с противоположной стороны, но размеры мира достаточно велики, чтобы он воспринимался как условно бесконечный.

Астероиды не являются одноразовой стартовой раздачей. Сервер поддерживает астероидное поле вокруг активных игроков:

- слишком дальние астероиды удаляются;
- новые астероиды появляются за пределами видимой зоны игрока;
- стартовая скорость направлена примерно к области игрока;
- при разрушении астероиды продолжают делиться на меньшие части.

## Комнаты

Сервер держит комнаты в памяти. Комната создается при первом подключении игрока. Для MVP этого достаточно, потому что цель — доказать сетевую механику, а не постоянное хранение мира.

## Ограничения текущего MVP

- Нет постоянного хранения сессий.
- Нет авторизации.
- Нет деплоя в интернет.
- Нет client-side prediction.
- Отрисовка использует примитивы Canvas.
- Сервер рассчитан на учебные комнаты 2-8 игроков.
